**Ver 1.0**
**Presented By:** CONNECTED TECH
**Date:** 8 June 2026

## Executive Summary

ปัจจุบัน Honda ไม่มีระบบติดตามตำแหน่งรถในลานจอด ทำให้ทีมงานต้องค้นหารถด้วยตนเอง เสียเวลาและเกิดความผิดพลาด โดยเฉพาะกับรถเสียที่ต้องการการดูแลเร่งด่วน

CONNECTED TECH พัฒนาระบบ AI Vision Tracking ที่ช่วยติดตามรถแต่ละคันตั้งแต่ออกจากไลน์ผลิต

ระบบสร้าง **'รหัสติดตามรถ (Tracking ID)'** อัตโนมัติ เพื่อผูกกับข้อมูลรถ ระบุสถานะ และบันทึกตำแหน่งแบบ Real-time ลงฐานข้อมูล

ทั้งหมดนี้ทำงานได้โดยไม่เพิ่มภาระให้พนักงานแม้แต่ขั้นตอนเดียว

---

## 01 Project Overview

### Pain Points: ปัญหาหลักในลานจอดรถใหม่

**1. ไม่รู้ตำแหน่งรถที่ชัดเจน**
เมื่อรถออกจากไลน์ผลิต ไม่มีระบบบันทึกตำแหน่ง ทีมงานต้องเดินสำรวจเอง เสียเวลาและเสี่ยงต่อความผิดพลาด

**2. แยกแยะรถได้ยาก**
รถทุกคันเหมือนกันและไม่มีป้ายทะเบียน ไม่สามารถใช้ระบบ LPR ทั่วไปได้ ทำให้ระบุตัวตนรถแต่ละคันไม่ได้

**3. ไม่ทราบสถานะ Real-time**
โดยเฉพาะ "รถเสีย" ที่ต้องการการดูแลเร่งด่วน ทีมช่างหาตำแหน่งไม่เจอ เพิ่ม Downtime ในกระบวนการ

### ทำไมถึงแก้ยาก? — Key Challenges

**1. ไม่มีป้ายทะเบียน**
ระบบ LPR ทั่วไปใช้ไม่ได้ ต้องสร้างวิธีระบุตัวตนรถแบบใหม่ทั้งหมด

**2. รถทุกคันเหมือนกันทุกประการ**
ไม่สามารถแยกแยะรถได้ด้วยสายตา ต้องสร้าง unique ID ใหม่ให้แต่ละคัน

**3. ต้องช่วยลดภาระของพนักงานได้จริง**
Solution ต้องทำงาน fully automated เพื่อลดภาระการทำงานของพนักงาน

**4. ลานจอดอาจมีขนาดใหญ่**
ต้องวางแผนการแบ่ง zone กล้องและ tracking ข้ามกล้องให้ครอบคลุม

## 02 Solution

### Solution Overview

ระบบทำงานขั้นตอนหลักแบบ **Fully Automated**

| ขั้นตอน | ชื่อ | รายละเอียด |
|---|---|---|
| 1 | **Vehicle Identification** | AI ตรวจจับรถที่จุด Start พร้อมสร้าง Tracking ID และผูกกับรหัสถังน้ำมันอัตโนมัติ — Output: บันทึกข้อมูลแบบ Real-time |
| 2 | **Status Detection** | กล้องที่ทางแยกจับทิศทางรถ ระบุสถานะ "รถดี" หรือ "รถเสีย" อัตโนมัติ ก่อนเข้าลานจอด |
| 3 | **Parking Lot Detection** | กล้องวิเคราะห์สถานะซองจอด (ว่าง/มีรถ) และบันทึกหมายเลขซองจอด (Parking ID) ลงระบบ Real-time |
| 4 | **Central Platform — Database & Dashboard** | ทีมงานค้นหารถได้ทันทีจาก 'รหัสถังน้ำมัน' รู้ตำแหน่งซองจอดและสถานะรถได้ทันที |

---

### Step 1: Vehicle Identification

**ระบบ AI ตรวจจับรถที่จุดเริ่มต้น**

**Honda Car Tracking System Architecture:**

`กล้องจุด Start (ระบบตรวจจับรถ + ดึงรหัสถังน้ำมัน)` → `AI Tracking Engine` → `Database (บันทึก Tracking ID + Oil Tank ID)`

**วิธีดึง oil_tank_id:**
- **Option A:** OCR อ่านจากกระดาษรหัสถังน้ำมันที่วางหน้ากระจกรถ
- **Option B:** ดึงข้อมูลตรงจาก PLC ของไลน์ผลิต *(แนะนำ)*
- ผูก 'รหัสติดตามรถ' เข้ากับ 'รหัสถังน้ำมัน' และบันทึกลงระบบอัตโนมัติ

#### Option A: OCR — อ่านรหัสถังน้ำมัน

**1. วิธีการทำงาน**
กล้องอ่านกระดาษรหัสถังน้ำมันที่วางไว้หน้ากระจกรถ Honda เพียงย้ายตำแหน่งมาด้านหน้ารถ

**2. สิ่งที่ Honda ต้องเตรียม**
ย้ายกระดาษรหัสถังน้ำมันมาวางหน้ากระจกรถให้กล้องมองเห็นชัด

✅ **ข้อดี:** ติดตั้งรวดเร็ว ลงทุนต่ำ และไม่ต้องดัดแปลงระบบ IT เดิม

⚠️ **ข้อควรระวัง:** เสี่ยงต่อ Human Error และสภาพแสงมีผลต่อความแม่นยำ

#### Option B: PLC Integration — ดึงข้อมูลโดยตรง

**1. วิธีการทำงาน**
ดึงรหัสถังน้ำมันตรงจาก PLC ของไลน์ผลิต แม่นยำ 100% และสามารถดึง status รถ (ดี/เสีย) ได้ในขั้นตอนเดียวกัน ทำให้ไม่ต้องใช้กล้องทางแยกระบุสถานะอีกต่อไป

**2. สิ่งที่ Honda ต้องเตรียม**
เปิดสิทธิ์เข้าถึง PLC และเชื่อมต่อข้อมูลเข้า database ของระบบ

✅ **ข้อดี:** แม่นยำ 100% และสามารถรู้ "สถานะรถ (ดี/เสีย)" ได้ทันทีตั้งแต่จุด Start

⚠️ **ข้อควรระวัง:** ต้องผ่านขั้นตอนขออนุมัติสิทธิ์เข้าถึงข้อมูล (IT Security) และใช้เวลา Setup ระบบนานกว่าแบบแรก

**Flow:** `Production Line` → `PLC (Oil Tank ID + Status: GOOD/DEFECT)` → `API` → `Database`

---

### Step 2: Status Detection

**ระบบระบุสถานะรถ (ดี/เสีย) อัตโนมัติ**

**1. วิธีที่ระบบระบุสถานะรถ**
- กล้องที่ทางแยกตรวจจับทิศทางการเดินรถ
- เส้นทางซ้าย = ลานรถดี (Good Vehicle Zone)
- เส้นทางขวา = ลานรถเสีย (Defect Vehicle Zone)
- ระบบ tag status ลง database ทันที

**2. กรณีใช้ PLC Integration**
ดึง status ได้โดยตรงจาก PLC
✓ ไม่ต้องพึ่งกล้องทางแยก (เพิ่มความแม่นยำและลดขั้นตอนการประมวลผลภาพ)

---

### Step 3: Parking Lot Detection

**ระบุตำแหน่งรถบนลานจอดพร้อมจัดเก็บ ID**

**ระบบตรวจจับซองจอดอัจฉริยะ — การระบุตำแหน่งอัตโนมัติ**

กล้องในลานจอดรถทำงานด้วยระบบ Smart Parking Detection เพื่อระบุหมายเลขซองจอดและบันทึกลงระบบทันที

- ระบุหมายเลขซองจอดแม่นยำ (เช่น A01, B12)
- บันทึกข้อมูลเข้า Database แบบ Real-time
- รองรับการแสดงผลบน Visual Map Dashboard

---

### Dashboard & Output

**1. สิ่งที่ทีมงาน Honda จะเห็นและใช้งานได้จริง**

Dashboard จะแสดงผลข้อมูลรถแบบ Real-time ครอบคลุมทั้งรหัสถังน้ำมัน ตำแหน่งซองจอด สถานะรถ และเวลาบันทึก ช่วยให้ทีมงานสามารถค้นหารถจาก 'รหัสถังน้ำมัน' ได้ทันที

**2. ฟีเจอร์หลักของ Dashboard**
- ฟีเจอร์การกรองตามสถานะรถ (ดี/เสีย)
- ค้นหาด้วยรหัสถังน้ำมัน
- ดูแผนผังลานจอดแบบจำลองภาพ (Visual Map)

*ตัวอย่างหน้าจอ Dashboard แสดงรายการรถ (Oil Tank ID, Tracking ID, Parking Lot, Parking Slot, Status, Time) และแผนที่โรงงาน/ลานจอดแบบ Real-time*

---

### Camera Placement Plan

**โซน 1: จุดเริ่มต้น (Start Point)**
ตรวจจับรถออกจากไลน์ผลิต เพื่อสร้าง Tracking ID และอ่านรหัสถังน้ำมัน

**โซน 2: เส้นทางเดินรถ (Transit Route)**
ติดตั้งกล้องมุมซ้อนทับ (Overlap) ตลอดเส้นทาง เพื่อให้ AI ติดตามรถได้ต่อเนื่อง

**โซน 3: ทางแยกคัดแยก (Sorting Point)**
ตรวจจับทิศทางเลี้ยว เพื่อระบุสถานะรถ (ดี/เสีย) อัตโนมัติ

**โซน 4: ลานจอดรถ (Parking Area)**
ระบุหมายเลขซองจอด (Smart Parking Detection)

## 04 Project Impact

### Benefits: คุณประโยชน์ของระบบ

**1. ลดเวลาค้นหา: ชั่วโมง → วินาที**

ระบบทำงานเองทั้งหมด พร้อมเชื่อมต่อระบบ ERP หรือ CMMS ได้ในอนาคต
